# 塔外智能官网重设计：从企业展示页到企业智能体入口

## 1. 业务理解

塔外智能的核心方向是企业智能体应用产品与场景案例。它不是做一个泛聊天机器人，而是让智能体进入真实业务场景，成为可执行、可复核、可沉淀的数字工作单元。

## 2. 设计定位

本作品将官网定位为“塔外智能 TowerOS 企业智能体应用平台官网”。它既要建立企业级品牌信任，也要承担智能入口的作用：访客可以描述行业、岗位或业务痛点，获得智能体方案建议。

## 3. 视觉策略

新版视觉弱化黑色赛博和强 AI 感，改为浅色企业科技风：白底、浅蓝灰、企业蓝、细网格、业务中枢图形。整体更接近大公司 B2B 官网，强调可信、清晰、专业和可落地。

## 4. 素材贴合

页面内容围绕产品介绍素材中的关键词展开：企业智能体应用产品、真实业务场景、岗位任务、工业安全、运营复盘、文档生成与信息处理，以及“可执行、可复核、可沉淀”的核心价值。

## 5. 信息架构

- 首页：建立品牌信任，表达 TowerOS 企业智能体平台定位。
- 产品体系：岗位任务智能体、企业知识智能体、业务流程智能体。
- 场景案例：工业安全、运营复盘、文档生成、岗位任务矩阵。
- 方案顾问：通过对话推荐智能体方案并收集线索。
- 作品说明：解释业务理解、视觉策略和技术实现。

## 6. 技术实现

- 后端：FastAPI
- 页面：Jinja2 + HTML + CSS + JavaScript
- 数据：JSON 知识库 + SQLite 线索表
- Agent：规则流 + 关键词检索，后续可升级为大模型 API 与向量知识库
